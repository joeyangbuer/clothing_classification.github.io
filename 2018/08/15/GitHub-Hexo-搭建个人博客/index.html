#GitHub + Hexo搭建个人博客

  在计算机领域，不论你是Developer还是Researcher,学习新知识和新技术已成为家常便饭，无论是环境的搭建、项目的开发、理论的学习以及论文的阅读，刚接触的时候记忆深刻，时间长了便都成了过眼云烟，为了避免陷入短期记忆快感的陷阱，必然少不了日常的总结和反思，写博客便成了不二的选择。众所周知，现在已经有很多网站能很方便支持大家的博客写作，常见的IT博客社区如CSDN、博客园等，这类博客主要以技术贴为主，界面画风较为单一；除此之外，要是你是个清新脱俗的程序员，在做一个技术宅同时又是一个文艺青年，喜欢码code空闲撸撸文章，那么简书也是个不错的选择（个人觉得一般简书上博客可读性更强，因为界面画风清新且排版清晰）。但作为长期混迹在全球第一大同性交流社区的人来说，当然得为github打call，毕竟github上搭建博客可拓展空间更大，更个性化，能避免博客网站的种种限制。本文主要介绍利用Hexo框架在GitHub上搭建个人博客的方法。

整体流程

  1. 安装Git
  2. 注册GitHub账号 并创建一个repository
  3. PC端安装Node.js
  4. 安装Hexo
  5. 推送博客至GitHub
  6. 绑定新域名
  7. 更换主题

具体步骤

第一步 安装Git

Git是个开源的分布式版本管理系统，可以敏捷高效的处理各种项目。GitHub是一个远程仓库，方便多人协作完成项目或共享项目，大家可以从GitHub上下载和完善你的代码。
安装包下载地址：git地址

完成安装之后，就可以使用命令行的 git 工具，在开始菜单里找到”Git”->”Git Bash”，会弹出 git 命令窗口，你可以在该窗口进行 git 操作。

打开界面后如图：

安装成功后，将你的git与github帐号绑定，。在命令行中输入 

git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"

紧接着生成ssh密钥文件：

ssh-keygen -t rsa -C "你的GitHub注册邮箱"

然后一直回车即可，默认不需要设置密码。找到生成的.ssh的文件夹中的id_rsa.pub密钥，将内容全部复制

打开GitHub_Settings_keys页面，新建new SSH Key，输入title（自拟）和key（将刚才复制的key粘贴至此）

在Git Bash中检测GitHub公钥设置是否成功，输入ssh git@github.com：

如上则说明成功。因为git和github间传输使用ssh加密，ssh是非对称加密算法，由一对公钥+私钥对来完成加解密（公钥加密 私钥可以解密，私钥加密 公钥可以解密），本地使用私钥加密传输到github端再用刚才添加的公钥解密。

第二步： 注册GitHub 创建一个新的repository

登录github官网点击sign up注册

注册成功后点击GitHub中的New repository创建新仓库，仓库名应该为：用户名.github.io 这个用户名使用你的GitHub帐号名称代替，这是固定写法，比如我的仓库名为：

第三步 PC端安装Node.js

去node.js官网下载安装Node.js，注意环境变量配置，win10应该是自动添加的，然后检测是否安装成功输入node -v

输入 npm -v检测node.js版本

如果上图所示则安装成功。

第四步： 安装Hexo

创建一个文件夹并右键点击Git Bash Here打开git bash（因为以下命令都在该目录下运行）
使用npm命令安装Hexo，输入： 

npm install -g hexo-cli

安装完成后 初始化我们的博客

hexo init blog

为了在本地测试我们的博客网站输入以下三条命令

hexo new g
hexo g
hexo s

执行完后，打开浏览器输入地址：localhost:4000

成功后会显示网页（下图显示是我修改了主题和标题后的图片）

如果网页未显示或者加载不出，解决方法见后。

第五步： 推送博客至github

进入主目录下的config文件

打开并编辑该文件修改以下内容，图中红框改为github上的仓库名称，如果按照我前面的设置就是账号名，最后保存配置文件

最后安装Git部署插件，输入命令：

npm install hexo-deployer-git –save

为了将博客部署至github仓库，输入接下来三条命令（最后两条也可以合成hexo g –d）：

hexo clean 
hexo g 
hexo d

部署成功后就可以用 用户名.github.io访问你的网站了

第六步： 绑定域名

虽然我们可以用已有的github的域名但是如果想要一个个性化的域名，那么就需要注册一个新的域名，我选择的是阿里云提供的域名服务，注册登录阿里云账号，下拉网页并点击“域名与网站”并选择“域名注册”，注册域名并按要求进行身份验证(身份验证可能需要等一段时间）

验证成功后，点击网页最上方导航栏的“控制台“，并在页面左边菜单中选中域名

然后在网页中对域名进行解析

然后添加两条记录，一条是域名间的映射（即新域名和github域名的映射，一条是和IP地址的映射，github域名对应的IP地址（可以通过在cmd中输入ping http://xxxx.github.io 得到域名对应的IP地址），然后填入进去

然后登录GitHub，进入之前创建的仓库，点击settings，设置Custom domain，输入你的域名

点击save保存。然后进入本地博客文件夹 ，进入source目录下，创建一个记事本文件，输入你的域名，对，只要写进你自己的域名即可。如果带有www，那么以后访问的时候必须带有www完整的域名才可以访问，但如果不带有www，以后访问的时候带不带www都可以访问。所以建议，不要带有www。这里我还是写了www(不建议带有www)，最后记得文件格式保存为所有文件而不是.txt文件。

完成这三步，进入本地博客目录中，右键打开Git Bash，依次输入：

hexo clean
hexo g
hexo d

这时候打开浏览器在地址栏输入你的个性化域名将会直接进入你自己搭建的网站。

第七步： 更换主题

点击hexo主题网址搜索,选择并点击你喜欢的主题的名称进入主题的github界面，复制顶部的url，在博客文件夹路径下执行（右键博客文件夹打开的bash,便自动在博客文件夹下，bash中可以看到当前路径，如果不在就cd进去）然后执行

git clone https://github.com/klugjo/hexo-theme-clean-blog （你喜欢博客主题github界面的 url）

此时你打开本地博客文件夹的themes目录会发现多了一个主题文件，即刚下载的主题

打开主目录下的_config.yml配置文件，修改主题为clean-blog

紧接着修改主题文件中的_config.yml配置文件，里面可以添加和设置你的导航目录，菜单名称等等，具体设置看你刚才所打开的主题的github界面下面有关配置的详细介绍,如图：

选择好后，依次输入

hexo g 
hexo d

重新部署网站并查看效果。

可能遇到的问题

Q1：输入localhost:4000没有响应
A1：端口被福昕阅读器的进程占用
如图用命令查看端口状态 发先localhos:4000被进程号（PID）为22908的进程占用

在任务管理器中查看进程号22908 发现被福昕的进程占用

解决方法一：右键点击服务并禁止
解决方法二：在本地测试hexo时候，用命令hexo s -p 5000 更换端口（此处更换为5000），然后用localhost:5000再访问

Q2： 执行hexo d后要求输入账号和密码 输入正确信息后在浏览器地址栏输入你的放置个人网站的仓库路径，即 http://xxxx.github.io 出现404错误
A2：把_config.yml文件中repository: https://github.com/xxxx/xxxx.github.io.git这个地址改为`git@github.com:xxxx/xxxx.github.io.git`(xxxx是在github上设置的仓库名称，一般是用户名)
